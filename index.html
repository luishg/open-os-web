Principles

Interoperability

Open formats, standard protocols, no vendor lock-in. Systems should communicate with each other. Data should move freely.

Local-first

User data lives on user devices by default. Cloud sync is optional, encrypted, and user-controlled.

Open Source

All code is public, auditable, and reusable under permissive licenses.

LLM Browser Extension

Released

Browser interface for local language models

A lightweight extension that connects to any LLM running on Ollama (Llama, Mistral, Phi, Gemma, and others). Chat with AI directly from your browser without external servers, accounts, or telemetry.

Available for Chromium browsers (Chrome, Brave, Edge).

Soul Protocol

In Progress

Specification for portable AI identities

A standard format to store AI assistant configuration, memory, and context in a way that can migrate between different models and providers.

Identity + Memory + Context
            ↓
    Portable across LLMs
    Local storage
    Optional sync

Your relationship with an AI assistant shouldn't be locked to a single provider. Soul Protocol defines how to make it portable.
